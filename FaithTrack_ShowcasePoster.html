Project Overview
What is FaithTrack?

Faith-based resources like Bible study guides, sermon notes, and prayer journals are typically scattered across multiple apps and documents with no central place to manage them. FaithTrack solves this by providing a secure, organized web platform where users can create, categorize, search, and manage all their faith resources in one place.

The project was developed across two semesters (CST-451 and CST-452) using an Agile Scrum methodology with three sprints, delivering all 20 user stories and 57 story points.

Key Features
What It Does
🔒 Secure Authentication
ASP.NET Core Identity with PBKDF2 password hashing, cookie sessions, CSRF protection on all POST routes, and role-based authorization scaffolding.
📚 Resource Management
Full CRUD for faith-based resources. Title, category, description, and optional URL. Delete requires explicit confirmation to prevent accidental data loss.
📁 Category System
Create and manage custom categories. Cascade-protection validation prevents deletion of categories that still have resources assigned.
🔍 Search & Filter
Keyword search by title and description. Filter resources by category for fast retrieval across large collections.
Engineering Challenges
Obstacles Overcome
Identity Integration
Wiring Identity into the N-Layer architecture without letting auth bleed into service or repository layers required careful middleware ordering and IdentityDbContext extension.
✓ Isolated to AccountController and application layer via DI
EF Core Migration Conflicts
Schema evolution across three sprints caused migration conflicts when code-first models diverged from the existing database state.
✓ Incremental migration strategy with local DB backup before each apply
Category Cascade Protection
Deleting a category with assigned resources violated the ON DELETE NO ACTION FK constraint. Needed meaningful user error, not a raw SQL exception.
✓ Pre-delete resource count check in CategoryService, error returned to view
